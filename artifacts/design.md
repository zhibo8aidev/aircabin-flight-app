# AirCabin Design

## 1. 设计目标
- 为 Android 首发版本提供适合机上弱网、离线、低光环境的高保真移动端设计基线。
- 在 3 秒内让用户识别当前是否有航班、当前处于什么飞行阶段、是否为模拟航线或实时修正状态。
- 用清晰的层级承载三条主线：当前航班体验、飞行统计沉淀、机上碎片时间整理相册。
- 通过匿名、封闭、低打扰的聊天室设计控制社交风险，不把产品导向开放式陌生人社交。
- 通过高对比深色主题、舱窗视角背景和航空仪表化信息卡建立科技感与航空出行感。

## 2. 信息架构

### 2.1 一级导航
- 首页：当前航班摘要、快捷功能、年度摘要、最近记录入口
- 航班：当前航班地图、状态、时间轴、登机信息与关联动作
- 统计：本次/月度/季度/全年统计、图表、年度报告入口
- 相册：本地媒体扫描、分类整理、批量删除
- 我的：飞行档案、权限与隐私、设置

### 2.2 二级信息结构
- 首页
  - 当前航班卡
  - 功能入口矩阵
  - 年度摘要
  - 历史航班入口
- 航班
  - 航线视图
  - 状态卡
  - 时间卡
  - 相关入口
- 统计
  - 时间维度切换
  - 核心指标
  - 趋势图表
  - 热门航线/城市
  - 年度报告
- 相册
  - 空间概览
  - 分类筛选
  - 工具栏
  - 媒体网格
  - 批量操作栏
- 聊天室
  - 房间头部信息
  - 规则公告
  - 话题标签
  - 消息流
  - 输入与发送反馈
- 我的
  - 用户卡片
  - 飞行档案入口
  - 权限与隐私
  - 设置

### 2.3 核心任务路径
- 导入登机信息 -> 确认字段 -> 生成当前航班卡 -> 查看航班页
- 首页当前航班卡 -> 航班页 -> 查看模拟航线/实时修正状态 -> 进入统计或聊天室
- 首页/相册页 -> 分类筛选 -> 多选预览 -> 系统确认删除 -> 空间释放反馈
- 统计页 -> 切换时间维度 -> 查看年度报告入口
- 我的页 -> 飞行档案 -> 历史记录与年度报告

## 3. 设计基调
- 主题：深色舱内模式，强调低亮度下可读性与沉浸感。
- 视觉关键词：夜航、舷窗、航电面板、冷色 HUD、云层流光。
- 主色：`#07111F`、`#0C1D31`、`#112B45`
- 高亮色：`#79E8FF` 用于飞行状态、主操作、图表强调
- 辅助色：`#8AB4FF` 用于信息标签，`#7EF7C8` 用于成功反馈，`#FFB86B` 用于警示，`#FF6D7A` 用于风险操作
- 字体建议：标题偏紧凑几何风，正文使用 Android 默认无衬线替代；数字采用等宽风格呈现仪表感

## 4. 页面逐页说明

### 4.1 首页
**页面目标**
- 让用户快速确认当前是否在飞、飞到哪、还有多久到达，以及还能继续做什么。

**结构说明**
- 顶部品牌栏显示“云舱”与消息/设置入口。
- 当前航班主卡采用大面积航线面板，呈现出发地、目的地、阶段、已飞/剩余/ETA。
- 主卡顶部右侧固定放置状态芯片：
  - `模拟航线`：仅基于计划数据和本地时间轴推演
  - `实时修正`：已接入网络校正后的状态
- 快捷功能采用 2 x 2 卡片：本次飞行统计、相册整理、同舱聊天室、年度飞行报告。
- 下方为年度摘要卡和历史航班入口。

**视觉重点**
- 当前航班卡优先级最高，采用舱窗轮廓和发光轨迹线。
- 通过微弱星点与经纬网背景构建科技感，但不覆盖主要数据可读性。

### 4.2 航班页
**页面目标**
- 承载“飞到哪了、什么阶段、多久到”的核心认知。

**结构说明**
- 地图容器占首屏约 40%，默认 2D 视图，3D 作为预留视觉开关。
- 地图角落必须长期显示说明文案：`当前位置为模拟估算，仅供飞行记录参考`。
- 顶部状态条同时显示：
  - 当前数据模式
  - 最近修正时间或本地推演时间
  - 网络状态
- 飞行状态卡展示阶段、速度、海拔、剩余航程、ETA。
- 时间进度卡展示总时长、已飞、剩余、计划起飞、预计到达。
- 底部快捷入口：聊天室、飞行统计、登机信息。

**视觉重点**
- 航线折线和飞机图标采用冷光效果；模拟模式用虚线与蓝青色，实时修正用实线与更亮的青色。
- 飞行阶段以徽标标签强化，如 `巡航中`、`下降中`。

### 4.3 统计页
**页面目标**
- 帮助用户理解当前与长期飞行数据，支撑“个人飞行档案”认知。

**结构说明**
- 顶部采用分段切换：本次、月度、季度、全年。
- 首屏核心指标卡采用横向滑动仪表卡。
- 图表区域包括趋势折线、航司分布环图、月份柱图。
- 下方为高频航线与城市分布列表。
- 页尾放年度报告 CTA。

**视觉重点**
- 图表样式统一使用深底高亮线，弱化网格线，避免机上屏幕眩光。
- 统计数字保持较大字重，突出“次数、小时、公里”三类核心值。

### 4.4 相册整理页
**页面目标**
- 在机上碎片时间内快速完成筛选、预览、删除，降低误删风险。

**结构说明**
- 头部空间卡显示可释放空间、候选数、扫描时间。
- 分类卡横滑展示：截图、大视频、相似图、连拍、旅途当天。
- 工具栏提供全选、排序、滑选模式。
- 3 列网格保持高密度浏览，视频缩略图显示时长，选中态显示明显描边与勾选。
- 底部悬浮操作栏只保留 `预览`、`删除`、`取消选择` 三个高频动作。

**视觉重点**
- 相册页延续深色主题，但内容区域略抬高亮度，避免缩略图过暗。
- 删除相关元素统一使用暖色警示，形成与飞行状态高亮的明确区分。

### 4.5 聊天室页
**页面目标**
- 为同航班同舱位用户提供有边界的轻量交流空间。

**结构说明**
- 顶部房间头显示航班号、舱位验证、在线人数。
- 公告卡必须长期可见，提醒匿名规则、文明发言和审核机制。
- 话题标签用于降低首条消息成本，如 `拼车`、`转机`、`目的地建议`。
- 消息流采用左对齐匿名气泡，自己的消息右对齐。
- 输入区旁边展示网络状态；无网时替换为草稿提示和禁用发送。

**视觉重点**
- 社交区域不做娱乐化，避免过度活跃氛围；使用规整、冷静的气泡和信息分隔。
- 风控反馈使用显著但不惊吓的橙红色提示。

### 4.6 我的页
**页面目标**
- 管理飞行档案、权限和个人偏好，提供信任与隐私控制入口。

**结构说明**
- 顶部个人卡显示昵称、年度飞行次数、飞行等级预留位。
- 飞行档案模块提供历史航班与年度报告入口。
- 权限与隐私分组展示相册权限、通知权限、数据同步、清除本地缓存。
- 设置区包含主题、关于、帮助与反馈。

**视觉重点**
- 我的页视觉更稳，弱化动态背景，强化设置清晰度。
- 清除缓存等风险操作放在区块底部，并辅以二次确认说明。

## 5. 交互规则

### 5.1 导航与切换
- 底部导航始终固定，当前 tab 使用高亮图标和文字。
- 首页点击当前航班卡、快捷卡、年度摘要均进入对应一级或二级页面。
- 页面切换动画以 220ms 左右的水平滑动和淡入为主，避免炫技式过度动画。

### 5.2 模拟航线与实时修正规则
- `模拟航线` 与 `实时修正` 必须以不同状态芯片、不同轨迹样式、不同辅助文案同时区分。
- 任一页面只要出现航线、位置、ETA、速度等可能被理解为实时导航的数据，都必须伴随模式标识。
- 模拟模式下默认文案：
  - `基于计划航线与飞行阶段估算，不代表真实位置`
- 实时修正模式下默认文案：
  - `已结合最近网络数据修正，仍用于行程参考，不作为导航依据`
- 当网络断开且之前存在实时修正时，状态自动回落为 `离线沿用修正后时间轴`，并显示最近修正时间。

### 5.3 相册操作规则
- 首次进入相册页先做权限教育，再请求系统授权。
- 删除动作必须经过三层保护：选中反馈、预览确认、系统删除确认。
- 删除成功后展示释放空间反馈；失败项需列出原因，不得静默吞掉。

### 5.4 聊天室规则
- 首次进入房间必须弹出规则确认。
- 无网络、未验证舱位、房间关闭三种状态下均不可直接发送消息。
- 长按消息提供举报、复制、拉黑；举报后立即出现受理反馈。
- 命中风控的消息保留草稿文本，并给出可理解原因。

### 5.5 反馈与动效规则
- 航班阶段变化、扫描完成、删除成功、发送成功使用轻量 toast 或内联反馈，不使用阻断弹窗。
- 卡片和图表的发光效果采用弱动效，控制在低频呼吸或渐变移动，适配机上低功耗场景。

## 6. 组件规范

### 6.1 布局与间距
- 基准栅格：4dp
- 页面左右边距：16dp
- 卡片圆角：20dp
- 小型芯片圆角：999dp
- 底部导航高度：64dp

### 6.2 核心组件
- 顶部品牌栏：标题、次级入口、弱分隔线
- Flight Hero Card：航班号、起降城市、状态芯片、关键时间、CTA
- Status Chip：支持 `模拟航线`、`实时修正`、`离线`、`需确认`
- Metric Card：大数字 + 单位 + 对比文案
- Chart Panel：标题、筛选、图表、图例
- Album Category Card：图标、分类名、候选数、可释放空间
- Chat Bubble：匿名头像、昵称、时间、气泡内容、操作入口
- Setting Row：图标、标题、描述、箭头或开关

### 6.3 字体与层级
- 页面标题：28sp，半粗体
- 分区标题：18sp，半粗体
- 卡片主数字：24sp，粗体
- 正文：14sp
- 辅助说明：12sp
- 状态芯片：11sp，大写或短词形式

### 6.4 图标与插画
- 图标风格统一为线性 + 轻量面填充。
- 飞机、航线、云层、经纬网作为主要品牌识别元素。
- 空状态插画建议使用舷窗、虚线航线和整理托盘等航空语义，而不是通用空盒子。

## 7. 状态设计

### 7.1 首页
- 正常态：存在当前航班，展示主卡和快捷入口
- 空状态：提示导入登机信息
- 离线态：卡片显示 `离线模拟`
- 数据回退态：显示最近一次修正时间

### 7.2 航班页
- 模拟航线态：虚线轨迹 + 模拟提示文案
- 实时修正态：实线轨迹 + 最近修正时间
- 飞行模式离线态：隐藏刷新提示，保留本地时间轴
- 无航班态：提示从首页导入或查看历史航班

### 7.3 统计页
- 有数据态：展示本次与年度统计
- 空数据态：引导先导入登机牌
- 加载态：骨架卡 + 图表占位
- 报告已生成态：显示最近一次生成时间

### 7.4 相册页
- 无权限态：展示权限申请说明
- 扫描中：显示进度、预计耗时和后台继续说明
- 空分类态：展示“该分类暂无可整理内容”
- 多选态：底部批量操作栏浮出
- 删除结果态：展示释放空间和失败原因

### 7.5 聊天室页
- 可加入态：正常消息流与输入
- 网络不可用态：不可发送，仅保留草稿
- 房间关闭态：显示归档说明
- 审核拦截态：消息发送失败，保留文本
- 新手引导态：首次进入规则弹窗

### 7.6 我的页
- 默认态：展示飞行档案和设置
- 权限未开启态：显示弱提醒
- 清除缓存确认态：二次确认弹窗

## 8. 设计风险与评审要点

### 8.1 设计风险
- 模拟航线若视觉过强，用户可能误解为真实导航，需要强制文案与状态区分。
- 深色主题在低亮度机舱中更友好，但图表、缩略图、标签若对比不足会损伤可读性。
- 相册整理涉及高风险删除，流程一旦压缩过度会放大误删概率。
- 聊天室在无网、弱网、审核拦截场景下的反馈需要足够明确，否则容易被理解为系统故障。
- 首页信息密度较高，若年度摘要和快捷入口占比失衡，会削弱当前航班的核心认知。

### 8.2 评审要点
- 是否所有航线和位置相关界面都明确区分了 `模拟航线` 与 `实时修正`
- 首页首屏是否在 3 秒内完成“有无航班、飞行阶段、可执行操作”三件事传达
- 统计页图表在深色主题下是否足够清晰，是否兼顾机上低光环境
- 相册删除链路是否具备足够保护，不会诱导误删
- 聊天室是否体现封闭、匿名、低打扰原则，且风控反馈明确
- Android 390 x 844 与 360 x 800 两档下，关键模块是否都能无截断展示
