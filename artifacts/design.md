# AirCabin Design Revision

## 1. 设计目标
- 将整体气质继续收敛到“极简、高级感、年轻化”的国际化出行工具界面。
- 保持六个核心页面范围不变：首页、航班页、统计页、相册整理页、聊天室页、我的页。
- 进一步提高背景与卡片的明暗对比，让界面更干净、更轻盈，但仍保留真实可用的信息边界。
- 去除冗长免责段落，改为更短、更轻量的状态标签、caption 与 chip 表达，既不误导也不打断阅读。
- 在 390 x 844 与 360 x 800 设备上继续提升呼吸感、触控舒适度与高级感。

## 2. 第二轮修订方向

### 2.1 视觉风格
- 关键词：冷静、留白、洁净、年轻、旅行用品感、国际化。
- 视觉语言：更灰的背景承托更白的内容卡，减少“奶油白糊在一起”的感觉。
- 层级建立方式：优先依赖背景层次、卡片明度、字重和间距，而不是长说明或重装饰。
- 状态表达：从整段解释切换为 `状态标签 + 极短 caption + 时间戳/来源 chip`。

### 2.2 色彩系统
- 主背景：`#ECEEF1`
- 背景渐层亮部：`#F5F6F8`
- 背景渐层暗部：`#E4E7EB`
- 主内容卡：`#FFFFFF`
- 次级卡：`#FAFBFC`
- 卡片描边：`rgba(17, 17, 17, 0.08)`
- 强分隔线：`rgba(17, 17, 17, 0.14)`
- 主文本：`#111111`
- 次文本：`#5F6368`
- 弱文本：`#8A9099`
- 点题橙黄：`#F1B547`
- 橙黄浅底：`rgba(241, 181, 71, 0.16)`
- 深灰强调：`#262626`
- 离线提示：`#B96A3C`
- 风险操作：`#C46B43`
- 原则：背景必须明显比卡片灰，卡片必须明显接近纯白，避免页面层级发虚。

### 2.3 字体与排版气质
- 标题保持几何无衬线，大字号、紧凑字距、有限字重对比。
- 正文保持中性、人文、国际化的阅读气质。
- 数字使用规整 tabular lining 风格，保证时间、里程、统计排版稳定。
- 文案长度继续收短，优先名词化表达，减少句子感。

## 3. 信息架构

### 3.1 一级导航
- 首页：当前航班摘要、关键入口、年度摘要
- 航班：模式、航线背景、飞行状态、时间进度、相关动作
- 统计：本次/月度/季度/全年统计、图表、年度报告入口
- 相册：端侧扫描结果、分类整理、批量删除
- 聊天室：同航班同舱位匿名交流
- 我的：飞行档案、权限与隐私、设置

### 3.2 页面结构原则
- 每页首屏仅保留一个主目标模块，其他内容下沉为次级卡片或列表。
- 关键数据按“标题 -> 状态 -> 数值 -> 动作”组织，解释信息仅以轻量 caption 补充。
- 首屏主要区块控制在 2 到 4 个，杜绝连续堆叠小模块。
- 二级入口优先使用列表行、轻按钮或低对比 action card，不与主卡抢中心。

## 4. 布局原则

### 4.1 留白与密度
- 页面左右边距：24dp
- 页面顶部首区与首卡距离：24dp 到 28dp
- 卡片之间垂直间距：18dp 到 24dp
- 大卡内部上下留白：24dp
- 中小卡内部上下留白：20dp 到 22dp
- 同组信息行间距：10dp 到 14dp
- 标题与说明之间最小距离：6dp
- 模块密度规则：首屏最多 1 张 hero card + 2 张中卡；超过内容必须进入滚动区

### 4.2 触控舒适度
- 主按钮最小高度：54dp
- 列表行最小高度：58dp
- 芯片最小高度：32dp
- 底部导航点击区高度：74dp
- 横向筛选 chip 间距：10dp
- 悬浮操作条与底部导航之间保留至少 16dp 安全距离

### 4.3 国际化视觉偏好
- 默认左对齐排版，避免密集多列。
- 减少装饰性描边与说明性段落，更多使用轻标签、caption、时间戳。
- 背景、卡片、模块之间要有可感知的明度层次，形成“空气感”。

## 5. 排版规则

### 5.1 字级
- 页面标题：32sp / 600
- 页面副标题或轻导语：15sp / 400
- 模块标题：20sp / 600
- 核心数字：30sp / 600
- 状态标签 / chip：12sp / 600
- 正文：14sp / 400
- caption：12sp / 400

### 5.2 文案层级
- 一级信息只承载一个明确判断，例如“模拟航线”或“剩余 1h 18m”。
- 二级信息改用极短 caption，例如“Plan only”“Synced 15:02”“Offline cache”。
- 三级信息只保留时间戳、缓存状态、网络状态，不再出现长段免责说明。
- 除敏感风险操作外，单条辅助文案尽量控制在 12 个汉字或 3 个英文词以内。

## 6. 状态区分规范

### 6.1 三种模式定义
- `模拟航线`：仅基于计划数据与飞行阶段推演。
- `实时修正`：最近一次联网后修正了时间轴与位置参考。
- `离线回退`：当前无网络，继续沿用最近一次修正结果。

### 6.2 必须同时出现的识别要素
- 状态标签：固定命名 `模拟航线`、`实时修正`、`离线回退`，不可改写为模糊词。
- 轻量 caption：在出现位置、轨迹、ETA 的模块中，必须搭配短提示，不使用长段说明。
- 来源 / 时间 chip：例如 `Plan only`、`Synced 15:02`、`Offline cache`。
- 轨迹样式：
  - 模拟航线：浅灰虚线 + 橙黄点位
  - 实时修正：深灰实线 + 橙黄飞机点
  - 离线回退：灰色点划线 + 最近修正时间

### 6.3 推荐短文案
- 模拟航线：
  - 状态标签：`模拟航线`
  - caption：`计划参考`
  - 来源 chip：`Plan only`
- 实时修正：
  - 状态标签：`实时修正`
  - caption：`已同步`
  - 来源 chip：`Synced 15:02`
- 离线回退：
  - 状态标签：`离线回退`
  - caption：`离线延续`
  - 来源 chip：`Offline cache`

### 6.4 禁止项
- 不再使用类似“基于计划航线与飞行阶段估算，仅供飞行记录参考”的整句长说明。
- 不得仅靠颜色区分模式，必须同时有标签文字与时间/来源信息。
- 不得让地图或轨迹区域缺少任何模式识别线索。

## 7. 页面逐页调整

### 7.1 首页
**调整目标**
- 用更干净的 hero card 完成“当前航班、模式、剩余时间”的首屏判断。

**新布局**
- 顶部欢迎区继续简洁，仅保留品牌、日期与轻设置入口。
- 主卡展示：航线、阶段、剩余时间、状态标签、短 caption、一个来源 chip。
- 快捷入口维持 2 列，但卡片高度略增、文字更短、内部留白更大。
- 年度摘要维持 2 个核心数字 + 1 行轻说明。

**页面调整点**
- 删除长免责句，改为 `计划参考` 等短 caption。
- 主卡内最多展示 3 组指标。
- 年度摘要说明控制在一行到两行内。

### 7.2 航班页
**调整目标**
- 让用户先看到模式和来源，再理解地图与时间，不误读成导航界面。

**新布局**
- 地图上方先展示模式条：状态标签 + 来源 chip + 网络状态 caption。
- 地图区保持克制，只保留起终点、轨迹、飞机点与轻 caption。
- 下方两张宽卡分别承载“飞行状态”和“时间进度”。
- 相关入口继续为垂直列表按钮，但行高增加。

**页面调整点**
- 地图底部说明改为短 caption，例如 `Reference only`。
- 2D / 3D 切换进一步弱化。
- 离线回退状态下必须紧跟最近修正时间。

### 7.3 统计页
**调整目标**
- 维持大指标优先，并让离线/缓存状态表达更轻、更国际化。

**新布局**
- 顶部分段控件更宽、更松。
- 首屏仅保留 1 张大指标卡 + 2 张小指标卡。
- 图表卡片单独存在，每张图表只保留一个缓存时间 chip。
- 年度报告 CTA 保持浅橙色，但卡片更白、更轻。

**页面调整点**
- 删除完整句式说明，改为 `Offline cache`、`Cached 14:58`。
- 图表网格线再淡一点。
- 空数据态保留大留白。

### 7.4 相册整理页
**调整目标**
- 继续强化“审阅与整理”的气质，而非“强刺激删除”。

**新布局**
- 顶部摘要卡仅保留空间与候选数量，扫描时间降为短 caption。
- 分类筛选 chip 与工具栏拉开距离。
- 3 列网格保留，但单元间距继续增大。
- 底部双按钮浮层与内容区增加更多缓冲留白。

**页面调整点**
- `仅在本机处理` 保留，但作为轻标签或 caption。
- 删除按钮维持风险色，不使用高饱和红。
- 选中态依赖边框、角标、轻阴影，不压暗整张缩略图。

### 7.5 聊天室页
**调整目标**
- 维持封闭、可信、克制的交流氛围，同时减少规则说明的压迫感。

**新布局**
- 顶部房间卡整合航班号、舱位验证、在线人数与一句极短规则 caption。
- 话题 chip 保持中性低对比。
- 消息气泡外边距进一步增大。
- 输入区上方弱网提示改为短标签 + caption。

**页面调整点**
- 规则说明压缩为一句，例如 `封闭匿名，仅限同舱位`。
- 草稿保留说明改为短 caption。
- 不增加社交装饰性元素。

### 7.6 我的页
**调整目标**
- 让个人页更像高端工具应用的档案中心，减少说明堆积。

**新布局**
- 顶部个人卡保持轻量，重点是昵称、年度次数与一句偏好标签。
- 飞行档案摘要卡继续承接历史航班与报告状态。
- 权限与隐私、设置继续分组，但行间距与组间距增大。
- 风险项继续独立靠后。

**页面调整点**
- 列表项状态值尽量短，例如 `On`、`Off`、`Wi-Fi only`。
- 不再补充长说明文案。

## 8. 组件规范

### 8.1 核心组件
- Hero Flight Card：航线大标题 + 状态标签 + caption + 来源 chip + 三组主指标 + CTA
- Mode Banner：状态标签 + 来源 chip + 网络 caption
- Metric Card：主数字 + 短状态
- Soft Action Card：标题、数值、轻 caption
- Filter Chip：中性灰底，选中态为浅橙底
- Privacy Notice：用于相册、聊天室的端侧 / 封闭边界说明，长度尽量控制为一句
- Setting Row：标题、短状态值、箭头

### 8.2 圆角与阴影
- 大卡圆角：30dp
- 小卡圆角：24dp
- 列表行圆角：20dp
- 芯片圆角：999dp
- 阴影：低透明、大半径、低高度差，不使用发光式阴影

## 9. 交互规则

### 9.1 页面切换
- 页面切换动效维持在 180ms 到 220ms，偏淡入与轻微位移。
- 不使用旋转、波纹、发光等炫技动效。

### 9.2 状态反馈
- 重要反馈优先内联显示，避免打断。
- 风险动作使用二次确认与结果页，视觉上克制但边界清晰。

### 9.3 可达性
- 文本与背景对比度不低于 AA。
- 所有关键按钮和列表行满足 44pt 以上点击面积。
- 状态标签、caption、时间 chip 必须协同出现，不能只靠颜色。

## 10. 本轮评审重点
- 背景更灰、卡片更白之后，页面层级是否更清楚、更高级。
- 长说明被压缩为状态标签、chip、caption 后，是否仍然不误导。
- 六页是否在不改功能范围的前提下，继续降低拥挤感并提升呼吸感。
- 首页与航班页的模式区分是否足够国际化、轻量、清晰。
- 相册页与聊天室页的边界提示是否更短，但仍可信、稳妥。
