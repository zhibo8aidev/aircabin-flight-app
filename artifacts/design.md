# AirCabin Design Revision

## 1. 设计目标
- 将整体气质从“深色科技 HUD”调整为“极简、高级感、年轻化”的国际化出行工具界面。
- 保持原有功能范围不变，继续覆盖：首页、航班页、统计页、相册整理页、聊天室页、我的页。
- 用更松弛的布局和更明确的信息优先级，让海外用户在首屏内快速识别当前航班、模式状态与下一步动作。
- 在任何涉及航线、位置、ETA、速度的页面中，持续明确区分 `模拟航线`、`实时修正`、`离线回退`，避免误导为真实导航。
- 让界面在 390 x 844 与 360 x 800 设备上都保持良好的留白、触控舒适度与模块呼吸感。

## 2. 修订方向

### 2.1 视觉风格
- 关键词：极简、克制、清洁、年轻、国际化、高级旅行用品感。
- 视觉语言：从“仪表盘式信息堆叠”转为“白底内容卡 + 大块留白 + 稳定分区”。
- 背景氛围：冷白底色中引入非常轻的灰阶渐层与橙黄色点题，不再使用高饱和发光效果。

### 2.2 色彩系统
- 主背景：`#F3F4F6`
- 页面渐层白：`#FBFBFA`
- 主文本黑：`#111111`
- 次文本灰：`#626262`
- 分隔线灰：`#D9DADD`
- 卡片底色：`#FFFFFF`
- 重点橙黄：`#F3B33E`
- 深灰强调：`#262626`
- 风险提示：`#D96B3B`
- 成功/稳定提示不再使用绿色高亮，统一采用中性灰黑文案与橙黄色点缀，保证整体克制。

### 2.3 字体与排版气质
- 标题风格：几何无衬线，大字号、紧凑字距、轻重量差对比。
- 正文字体：稳定中性的人文无衬线，强调长时间阅读舒适度。
- 数字呈现：使用规整的 tabular lining 风格，提升时间、里程和统计信息的整洁感。

## 3. 信息架构

### 3.1 一级导航
- 首页：当前航班摘要、关键入口、年度摘要
- 航班：航线视图、状态数据、时间轴、相关动作
- 统计：本次/月度/季度/全年统计、图表、年度报告入口
- 相册：端侧扫描结果、分类整理、批量删除
- 聊天室：同航班同舱位匿名交流
- 我的：飞行档案、权限与隐私、设置

### 3.2 页面结构原则
- 每页首屏只保留一个主目标模块，其他内容降为次级卡片或列表。
- 关键数据按“标题 -> 状态 -> 解释 -> 行动”组织，不再并列堆满多组指标。
- 卡片数量控制在首屏 2 到 4 个主要区块，降低拥挤感。
- 二级入口尽量变为列表行或轻按钮，避免与主 CTA 争抢注意力。

## 4. 布局原则

### 4.1 留白与密度
- 页面左右边距：20dp
- 页面顶部首区与首卡距离：20dp 到 24dp
- 卡片之间垂直间距：16dp 到 20dp
- 卡片内部上下留白：20dp
- 同组信息行间距：8dp 到 12dp
- 模块密度规则：首屏最多 1 张大卡 + 2 张中卡；超过部分必须下沉到折叠或滚动区域

### 4.2 触控舒适度
- 主按钮最小高度：52dp
- 列表行最小高度：56dp
- 芯片最小高度：32dp
- 底部导航点击区高度：72dp
- 水平滑动筛选芯片之间保留 8dp 间距，避免误触

### 4.3 国际化视觉偏好
- 默认左对齐排版，避免多列细碎信息块。
- 更少的描边与更浅的装饰，主要依赖字号、字重、留白建立层级。
- 使用大卡片、短段落、明确按钮文案，让页面更接近国际旅行与生活方式产品的界面感受。

## 5. 排版规则

### 5.1 字级
- 页面标题：32sp / 600
- 页面副标题或导语：15sp / 400
- 模块标题：20sp / 600
- 核心数字：30sp / 600
- 卡片标签：12sp / 500
- 正文：14sp / 400
- 辅助说明：12sp / 400

### 5.2 文案层级
- 一级信息只承载一个判断，例如“当前是模拟航线”或“剩余 1h 18m”。
- 二级说明承担免责与来源解释，例如“基于计划航线估算，不代表真实位置”。
- 三级信息用于时间戳、缓存时间、网络状态，不与主数字抢权重。

## 6. 模式区分规范

### 6.1 三种模式定义
- `模拟航线`：仅基于计划数据、本地时间轴与飞行阶段推演。
- `实时修正`：最近一次联网后对时间轴与位置进行了修正，但仍仅用于行程参考。
- `离线回退`：曾存在实时修正，当前无网络，继续沿用上一次修正结果向后推演。

### 6.2 必须同时出现的识别要素
- 状态芯片：三种模式使用固定命名，不可替换为含糊词汇。
- 说明文案：每次出现位置、航线、ETA 时，必须有“仅供记录/参考，不作为导航”的说明。
- 轨迹样式：
  - 模拟航线：浅灰虚线 + 橙黄点位
  - 实时修正：深灰实线 + 橙黄飞机点
  - 离线回退：灰色点划线 + “最近修正时间”提示

### 6.3 默认文案
- 模拟航线：`基于计划航线与飞行阶段估算，仅供飞行记录参考`
- 实时修正：`已结合最近网络数据修正，仍不代表真实导航位置`
- 离线回退：`当前无网络，沿用最近修正结果继续推演，仅供参考`

## 7. 页面逐页调整

### 7.1 首页
**调整目标**
- 用更少模块完成“是否在飞、当前模式、还能做什么”的首屏判断。

**新布局**
- 顶部为简洁欢迎区，仅保留品牌、当前日期与一个轻量设置入口。
- 首屏主卡改为单一大卡：航线、阶段、剩余时间、模式芯片、免责说明集中展示。
- 原 2 x 2 快捷宫格改为 2 列低密度入口卡，卡片更高、更少字、更强留白。
- 年度摘要从多指标硬分栏改为 2 个核心数字 + 1 条说明，减少视觉拥堵。

**页面调整点**
- 起降机场采用大字排版，中间箭头弱化。
- 主卡中最多展示 3 组关键指标，其他信息通过次级列表承载。
- 历史航班入口下沉为列表行，不再与当前航班卡争抢首屏。

### 7.2 航班页
**调整目标**
- 把地图从“信息展示区”变为“认知背景区”，重点让用户看懂模式与时间状态。

**新布局**
- 地图容器上方先展示模式条，包含模式芯片、最近修正时间、网络状态。
- 地图本体更干净，减少装饰，只保留起终点、飞机点位、轨迹与必要说明。
- 地图下方使用两张宽卡分别承载“飞行状态”和“时间进度”。
- 相关入口改为 3 个垂直列表按钮，提升触控舒适度。

**页面调整点**
- 说明文案固定在地图区底部，且对比度足够明显。
- 2D / 3D 切换缩小为辅助控件，不抢视觉中心。
- 离线回退状态下，最近修正时间必须紧跟模式条显示。

### 7.3 统计页
**调整目标**
- 从“多图表叠加”调整为“核心数据优先，图表做辅助”。

**新布局**
- 顶部切换条改为宽胶囊 segmented control，四项间距更充足。
- 首屏只保留 1 张大指标卡和 2 张小指标卡。
- 图表拆成单独卡片，每张图表拥有独立标题与缓存时间说明。
- 年度报告 CTA 作为底部醒目但不过度突出的浅橙色卡片。

**页面调整点**
- 弱化网格线与装饰色，保持中性灰图表基底。
- 高亮仅用于当前时间段或关键数据点。
- 空数据态时保留较大留白，避免“空白页面焦虑”。

### 7.4 相册整理页
**调整目标**
- 让批量清理更像“审阅与决策”而不是“高压删除工具”。

**新布局**
- 顶部摘要卡只保留可释放空间与候选数量两项主信息，扫描时间降为小字。
- 分类筛选采用横向浅底芯片，选中项使用橙黄色强调。
- 工具栏简化为“全选 / 排序 / 滑选”三组操作。
- 网格保持 3 列，但增大缩略图间距，降低拥挤感。
- 底部操作区采用悬浮双按钮：`预览` 与 `删除所选`，取消选择改到顶部工具栏。

**页面调整点**
- 删除按钮采用橙褐色风险强调，但不使用高饱和红。
- 选中态以边框和角标提示，不大面积压色。
- 明确标注“仅在本机处理”，加强隐私信任。

### 7.5 聊天室页
**调整目标**
- 强化封闭、克制、可信的交流氛围，避免娱乐化社交界面感。

**新布局**
- 顶部房间信息区整合航班号、舱位验证、在线人数。
- 公告卡固定靠前，规则与边界说明可一眼读完。
- 话题标签采用低对比中性色，减少社交产品感。
- 消息气泡增大外边距，拉开会话节奏。
- 输入区固定底部，网络波动时在输入区上方出现轻提示条。

**页面调整点**
- 自己的消息仍与他人区分，但不使用鲜艳颜色。
- 草稿保留、禁发说明与审核反馈采用统一中性提示样式。
- 禁止把聊天室误读为开放社区，因此不展示头像个性化装饰。

### 7.6 我的页
**调整目标**
- 让“飞行档案”和“权限设置”更像高端工具应用，而不是功能杂糅设置页。

**新布局**
- 顶部个人卡更轻，重点展示昵称、年度飞行次数和一句偏好描述。
- 飞行档案改为单独摘要卡，再用列表行承接历史航班与年度报告。
- 权限与隐私、设置分别为两个分组列表，增加行高和分割留白。
- 风险项独立置底，不与常规设置混排。

**页面调整点**
- 视觉上减少图标堆积，以文本和行距建立秩序。
- 同步策略、通知权限等信息改为简短状态值，避免过长说明。

## 8. 组件规范

### 8.1 核心组件
- Hero Flight Card：大标题航线 + 状态芯片 + 三组主指标 + 免责说明 + CTA
- Mode Banner：模式、最近修正时间、网络状态
- Metric Card：单指标主数字 + 一句解释
- Soft Action Card：图标、标题、数值、轻箭头
- Filter Chip：中性灰底，选中态为橙黄底
- Privacy Notice：用于相册、聊天室等敏感模块的端侧/规则说明
- Setting Row：标题、状态值、箭头

### 8.2 圆角与阴影
- 大卡片圆角：28dp
- 小卡片圆角：22dp
- 列表行圆角：18dp
- 芯片圆角：999dp
- 阴影：使用低对比、大半径、低透明度阴影，不使用发光外发散

## 9. 交互规则

### 9.1 页面切换
- 页面切换动效控制在 180ms 到 220ms，偏淡入与轻微位移。
- 不使用旋转、发光、波纹式视觉炫技。

### 9.2 状态反馈
- 重要反馈优先内联，不打断当前任务。
- 风险动作使用二次确认与结果页，不使用恐吓式红色弹窗。

### 9.3 可达性
- 文本与背景对比度不低于 AA 基线。
- 所有关键按钮和列表行都满足 44pt 以上点击面积。
- 模式芯片与说明文案不能只靠颜色区分，必须同时依赖文字。

## 10. 本次评审重点
- 新的冷白极简风格是否已经明显脱离旧版深色科技感。
- 六页是否在保持功能范围不变的前提下，显著改善了拥挤度、留白与层级。
- `模拟航线`、`实时修正`、`离线回退` 是否在首页与航班页均足够显性，且不会误导成真实导航。
- 相册页删除操作与聊天室弱网反馈是否仍然清晰、可信、稳妥。
