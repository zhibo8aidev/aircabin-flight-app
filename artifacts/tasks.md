# Tasks - 云舱 AirCabin

## 0. 交付边界
本任务清单基于已批准的 proposal/specs/design，目标是完成 Android 首发 MVP：支持登机信息导入、航线模拟、本次与年度飞行统计、本地相册整理、同航班同舱位聊天室，以及必要的隐私与风控能力。

---

## 1. 项目初始化与基础设施
- [ ] 创建 Android 项目骨架（Jetpack Compose）
- [ ] 配置模块结构：app / feature / core / data / domain
- [ ] 配置导航框架与全局 theme
- [ ] 配置 Room、DataStore、Retrofit、OkHttp、WorkManager、Coil
- [ ] 配置日志、崩溃监控、环境切换
- [ ] 配置基础 CI：lint / assemble / unit test

## 2. 领域模型与契约
- [ ] 定义 `FlightRecord`、`FlightSession`、`YearlyFlightSummary` 领域模型
- [ ] 定义 `MediaAsset`、`MediaSimilarityGroup` 本地模型
- [ ] 定义 `ChatRoom`、`ChatMessage`、`ReportTicket` 模型
- [ ] 固化 REST / WebSocket 契约
- [ ] 准备 mock 数据与开发环境 API stub

## 3. 登机信息导入
### 3.1 拍照 / 截图 / 手动录入
- [ ] 实现导入入口页
- [ ] 接入拍照、截图选择、手动录入
- [ ] 集成 OCR 识别链路
- [ ] 实现 OCR 结果确认页
- [ ] 支持低置信度字段高亮修正

### 3.2 航班记录落库
- [ ] 航班记录写入本地数据库
- [ ] 导入成功后生成当前航班卡片
- [ ] 支持历史航班列表查询
- [ ] 支持删除或编辑单条航班记录

## 4. 当前航班与航线模拟
### 4.1 首页当前航班卡片
- [ ] 展示航班号、起终点、阶段、剩余时间
- [ ] 展示状态 chip：模拟航线 / 实时修正 / 离线回退
- [ ] 接入从导入记录到首页的闭环

### 4.2 航班页
- [ ] 集成地图容器
- [ ] 绘制起点、终点、路线 polyline
- [ ] 实现当前模拟位置计算
- [ ] 实现 2D / 伪 3D 视角切换
- [ ] 展示速度、海拔、剩余航程、ETA
- [ ] 覆盖离线态与无当前航班空状态

### 4.3 飞行阶段算法
- [ ] 实现基于时间比例的阶段划分
- [ ] 实现 provider 可选修正能力
- [ ] 实现状态刷新与本地缓存回退逻辑

## 5. 飞行统计与年度报告
### 5.1 本次飞行统计
- [ ] 展示已飞、剩余、总时长

### 5.2 历史与年度统计

## 实施顺序建议
1. 先完成项目骨架、模型与契约
2. 先打通导入 -> 当前航班 -> 统计主链路
3. 再完成相册整理能力
4. 再接入聊天室与风控
5. 最后完成测试、灰度、发布准备

## 角色拆分建议
### Android
- 首页 / 航班 / 统计
- 相册整理
- 聊天室 / 设置

### Backend
- 航班记录 / 统计 / 聊天 / 举报风控

### QA
- 弱网、离线、权限、相册删除、聊天室治理专项
